  ---
  关键改进点（解决你提到的问题）

  1. 招标人提取不准确 → 用 deer-flow 的 Skills 编写详细的字段提取指南 + Memory 记录修正历史
  2. 招标正文信息不完全 → 用 Sub-agent 并行处理：一个提取基础字段，一个专门分析 PDF 附件，一个做字段补全
  3. 前端展示问题 → deer-flow 的 ThreadState 支持结构化输出，前端可以直接消费标准化的 TenderNoticeSchema
